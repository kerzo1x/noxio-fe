1. Единый UI Kit (Базовые компоненты)
Вместо копипасты верстки мы создаем «кирпичики», которые сами знают, как себя вести.

BaseInput.vue: Один компонент на все случаи. Умеет показывать лейбл, обрабатывать ошибки, менять тип (text/email/password) и сам рендерит «глазик» для пароля через v-if="type === 'password'.

BaseButton.vue: Принимает isLoading, блокируется при отправке и меняет текст/показывает спиннер.

BaseMessage.vue: (В планах) Универсальная плашка для вывода ошибок бэкенда или успеха.

2. Глобальная чистка стилей (Tailwind DRY)
Выносим повторяющиеся классы (типа .field-input, .btn-primary) в src/assets/styles/main.css через директиву @layer components { @apply ... }.

В самих компонентах оставляем только уникальную логику, чтобы HTML не превращался в «кашу» из 50 классов.

3. Реорганизация логики (API Layer)
Переходим от «дикого» феча внутри компонентов к системному подходу:

Axios: Создаем экземпляр с baseURL, чтобы не хардкодить https://hrica.skyro.dev/... в каждом файле.

Services: Выносим запросы в отдельные файлы (например, auth.service.ts). Компонент страницы должен просто сказать: auth.login(data), а не знать, какие там заголовки и методы.

TypeScript: Описываем интерфейсы ответов, чтобы забыть про ошибку implicitly has an 'any' type.

4. Наведение порядка в Git
Переименовываем ветку из «страшной» в понятную: refactor/modular-ui или refactor/v2-architecture.

Синхронизируем локальное имя с GitHub/GitLab через удаление старой ветки и пуш новой.

5. UX без костылей (Прощай, setTimeout)
Удаляем искусственные задержки. Если бэкенд быстрый (0.3с) — фронтенд должен летать.

Плавность через Vue Transitions: Вместо того чтобы заставлять юзера ждать 1.5 сек, делаем мягкий fade переход между логином и 2FA/главной.

Фидбек: Сообщение «Success» показываем либо моментально перед переходом, либо уже как приветствие на следующей странице.

1. Error Handling & Logger (Твой "Черный ящик")
Вместо того чтобы в каждом компоненте писать try { ... } catch (e) { console.error(e) }, мы сделаем централизованную систему.

Глобальный перехватчик (Axios Interceptor): Все ошибки от бэкенда (401, 500, 403) будут ловиться в одном месте. Если пришла 401 — автоматически кидаем на логин.

Logger Service: Создадим утилиту src/utils/logger.ts.

Зачем: В режиме разработки она спамит в консоль красиво, а в продакшене — может отправлять логи в Sentry или просто молчать.

2. Validation (Живой фидбек)
Чтобы инпут краснел сразу, а не после того, как бэкенд выругается.

Схема валидации: Будем использовать либо легкую самописную логику в BaseInput, либо библиотеку (VeeValidate/Zod).

UX: Если поле пустое или формат почты неверный — кнопка BaseButton остается задизейблена.

3. Enter (Клавиатурный ниндзя)
Native Form Submit: Мы уже обернули всё в <form>, так что Enter работает.

Автофокус: Добавим пропс autofocus в BaseInput, чтобы юзер мог начать печатать сразу, не трогая мышку.

4. Skeletons (Визуальная скорость)
Пока Axios ждет ответа от бэкенда 0.3с, мы не показываем пустой экран.

BaseSkeleton.vue: Создадим универсальный серый анимированный блок.

Зачем: Это создает ощущение, что приложение загрузилось мгновенно.

5. Toast Notifications (Всплывашки)
Вместо того чтобы пихать message в верстку каждой страницы, мы будем вызывать уведомление из любой точки кода.

Пример: toast.success('Login successful!') или toast.error('Invalid code').

Реализация: Либо через Pinia, либо через готовую библиотеку типа vue-sonner (очень стильная, как раз под твой темный дизайн).
Децентрализация валидации (isError)

Вместо одной общей isError используй объект состояний (например, const errors = ref({ email: false, password: false })).

Передавай в каждый инпут только его конкретный флаг. Это позволит подсвечивать красным только проблемное поле, а не всю форму разом.

Функциональный сброс ошибок

Вынеси логику обнуления сообщений из атрибутов шаблона в метод handleClearError(field).

Это уберет «инвалидные» комментарии // message='' из HTML и позволит одной строчкой сбрасывать и флаг, и текст ошибки.
